1. GNU nano

1.1. Introduction to GNU nano

  • Simple, user-friendly terminal-based text editor
  • Ideal for quick edits and configuration files
  • Includes all the basic functionality of a regular text editor
  • Available on most Linux/Unix systems: nano --version

1.2. Starting nano

  • Open terminal and type: nano [filename]
  • Creates new file if filename doesn't exist
  • Opens existing file for editing if it exists

1.3. Syntax highlighting

  • Nano ships with syntax highlighting rules for most popular file types
  • /usr/share/nano directory
  • If not working add include "/usr/share/nano/*.nanorc" to ~/.nanorc

1.4. Commands

  • All commands are prefixed with either ^ or M characters
  • ^ represents Ctrl key
  • M- represents Alt key
  • List all commands: Ctrl+g
  • cheatsheet
  • Bottom two lines show common commands

1.5. Basic Navigation

  • Use arrow keys to move cursor
  • Page Up/Down to scroll through longer documents
  • Home/End to jump to start/end of line
  • Ctrl+_ to move to specific line and character number

1.6. Editing Text

  • Just start typing to insert text
  • Backspace to delete characters
  • Enter to create new lines

1.7. Saving and Exiting

  • Ctrl+O to save ("Write Out")
  • Ctrl+X to exit
  • Prompts to save if there are unsaved changes

1.8. Cut, Copy, and Paste

  • Ctrl+K to cut entire line
  • Alt+A to set mark, move cursor to select text
  • Ctrl+U to paste cut/copied text

1.9. Search and Replace

  • Ctrl+W to search for text
  • Ctrl+\ to search and replace

1.10. Customizing nano

  • Create ~/.nanorc file for personal settings
  • Set options like line numbers, auto-indentation

1.11. Practice Exercise

  • Open a new file with nano
  • Type a few lines of text
  • Save the file and exit
  • Reopen the file and make some edits

1.12. Additional resources

  • GNU nano: Official Website
  • Online manual
  • GNU nano FAQ

2. RAPIDS.ai Ecosytem

2.1. What is RAPIDS?

  • GPU-accelerated data science libraries for Python
  • Developed by NVIDIA
  • Aims to speed up end-to-end data science and analytics pipelines

2.2. Key Components

2.2.1. cuDF

  • GPU DataFrame library
  • Similar API to pandas
  • cuDF documentation

2.2.2. cuML

  • Machine learning library
  • Accelerated versions of scikit-learn algorithms
  • cuML documentation

2.2.3. cuGraph

  • Graph analytics library
  • For analyzing large networks
  • cuGraph documentation

2.3. Benefits of RAPIDS

  • Significant speedup over CPU-based libraries
  • Seamless integration with existing data science workflows
  • Open-source and community-driven

2.4. Getting Started

  • Install RAPIDS
    • Conda
    • pip
    • Docker
  • Compatible with NVIDIA GPUs (Pascal architecture or newer)
  • Jupyter Notebook integration

2.5. Hardware limitations

  • Python 3.9, 3.10, or 3.11
  • CUDA 11.2, 11.8, 12.0, or 12.2

2.6. Example code: cuDF

import cudf

# Create a GPU DataFrame
gdf = cudf.DataFrame({'A': [1, 2, 3], 'B': [4, 5, 6]})
print(gdf.head())

   A  B
0  1  4
1  2  5
2  3  6

2.7. RAPIDS Ecosystem

  • Integration with other data science tools
  • Dask for multi-GPU and multi-node scaling
  • XGBoost GPU acceleration
